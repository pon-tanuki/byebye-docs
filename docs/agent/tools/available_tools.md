# AIエージェント利用可能ツール一覧

## 概要

本ドキュメントはAIエージェントが使用可能なツールとそのユースケースを定義する。

---

## ツールカテゴリ

### 1. コード生成・編集ツール

#### code_generator

コードを生成するためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 要件に基づいてコードを生成 |
| 入力 | 要件定義、言語、フレームワーク |
| 出力 | ソースコード |
| 制約 | 1回の生成は500行以内 |

**ユースケース:**
- 新規機能の実装
- ボイラープレートコードの生成
- テストコードの自動生成

#### file_editor

ファイルを編集するためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 既存ファイルの読み取り・編集 |
| 入力 | ファイルパス、編集内容 |
| 出力 | 編集後のファイル |
| 制約 | プロジェクト内のみ |

**ユースケース:**
- バグ修正
- リファクタリング
- 設定ファイルの更新

---

### 2. テスト・品質ツール

#### test_runner

テストを実行するためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 単体テスト・統合テストを実行 |
| 入力 | テスト対象、オプション |
| 出力 | テスト結果、カバレッジ |
| 制約 | タイムアウト: 10分 |

**ユースケース:**
- CI/CDパイプラインでのテスト実行
- 変更影響の検証
- 回帰テスト

#### linter

コード品質をチェックするためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 静的解析・スタイルチェック |
| 入力 | ソースコード、ルール設定 |
| 出力 | 警告・エラーリスト |
| 制約 | 言語別に対応 |

**ユースケース:**
- コーディング規約の遵守確認
- 潜在的バグの検出
- コードレビュー前のチェック

#### formatter

コードを整形するためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | コードフォーマット適用 |
| 入力 | ソースコード |
| 出力 | フォーマット済みコード |
| 制約 | 設定ファイルに従う |

**ユースケース:**
- 一貫したコードスタイルの適用
- コミット前の自動整形

#### security_scanner

セキュリティスキャンを行うためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 脆弱性・セキュリティ問題の検出 |
| 入力 | ソースコード、依存関係 |
| 出力 | 脆弱性レポート |
| 制約 | OWASP Top 10 対応 |

**ユースケース:**
- デプロイ前のセキュリティチェック
- 依存関係の脆弱性確認
- コードレビュー補助

### 3. バージョン管理ツール

#### git_client

Gitリポジトリを操作するためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | Git操作（clone, commit, push等） |
| 入力 | コマンド、パラメータ |
| 出力 | 操作結果 |
| 制約 | 許可されたブランチのみ |

**ユースケース:**
- コード変更のコミット
- ブランチ作成・マージ
- 履歴確認

**許可される操作:**
- `git clone`
- `git checkout -b`
- `git add`
- `git commit`
- `git push`（feature/*, fix/*, agent/*）
- `git pull`
- `git status`
- `git log`
- `git diff`

**禁止される操作:**
- `git push --force`
- `git reset --hard`（リモート影響あり）
- `git push origin main/master`（直接）

| 項目 | 内容 |

### 6. 通信ツール

#### notification_sender

通知を送信するためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 通知の送信（Slack、Email等） |
| 入力 | メッセージ、宛先、チャンネル |
| 出力 | 送信結果 |
| 制約 | 許可されたチャンネルのみ |

**ユースケース:**

- レビュー依頼

#### api_client

外部APIを呼び出すためのツール。

| 項目 | 内容 |
|------|------|
| 説明 | 許可された外部APIの呼び出し |
| 入力 | エンドポイント、パラメータ |
| 出力 | APIレスポンス |
| 制約 | ホワイトリストのみ |

**ユースケース:**
- 外部サービス連携
- データ取得
- Webhook送信

---

## ツール使用の原則

1. **最小権限**: 必要最小限のツールのみ使用
2. **監査対応**: すべての使用をログに記録
3. **エラー処理**: 失敗時は適切にハンドリング
4. **タイムアウト**: 長時間実行は分割または報告
